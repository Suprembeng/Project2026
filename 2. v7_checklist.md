# Audit Rubric · v7.0 · STEP_XX.md Checklist

> **Версия**: v1.0 (для CORE v7.0+)
> **Использование**: skill `audit-step-spec` v3.0 прогоняет все пункты.
> **Всего пунктов**: 60 (+10 условных, активируются по контексту).

**Типы проверок**:
- `regex_in_file` — grep по файлу.
- `yaml_field_equals` — YAML parse + сравнение значения.
- `ajv_schema_pass` — ajv validate против schema.
- `external_script` — Python/shell скрипт возвращает 0 или JSON.
- `llm_judgment` — Opus/Sonnet возвращает bool/score.

## АГРЕГАЦИЯ

```python
rubric_pass_rate = pass_count / (total_items - n_a_items)
severity_weighted_score = (P0_fail * 25) + (P1_fail * 10) + (P2_fail * 3) + (P3_fail * 1)

verdict = compute_verdict(rubric_pass_rate, severity_weighted_score)
```

**Пороги**:
- `pass` → severity_weighted_score ≤ 5 AND rubric_pass_rate ≥ 0.95
- `pass_with_warnings` → severity_weighted_score ≤ 20 AND rubric_pass_rate ≥ 0.90
- `changes_required` → severity_weighted_score ≤ 50
- `fail` → >50 или любой P0
